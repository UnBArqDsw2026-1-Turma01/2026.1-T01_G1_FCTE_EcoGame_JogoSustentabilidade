Durante as reuniões do processo desing sprint, foram elecitados pelos alunos alguns requisitos funcionais e não funcionais, nos quais estão descritos neste documento

## Requisitos funcionais 

| ID | Nome | Descrição |
| :--- | :--- | :--- |
| RF01 | Movimentação | Processar entradas WASD para deslocar avatar 2D. |
| RF02 | Pesquisa de Resíduos | Visualização de objetos coletáveis (lixo) no mapa. |
| RF03 | Coleta de Itens | Remover objeto do cenário e somar ao inventário após colisão/ação. |
| RF04 | Depósito/Reciclagem | Validar proximidade à lixeira para descarregar itens. |
| RF05 | Crafting (Geração) | Criar item útil/decorativo após atingir meta X de resíduos. |
| RF06 | Progressão de Fases | Gerenciar transição: Limpeza -> Replantio -> Reconstrução. |
| RF07 | Interação | O sistema deve permitir interação com objetos e cenários.|
| RF08 | Reciclagem | O sistema deve possuir um sistema de reciclagem (máquina única).|
| RF09 | novos recursos | O sistema deve possuir progressão baseada em desbloqueio de novos recursos.|
| RF10 |Ferramentas| O sistema deve permitir uso de ferramentas (machado, picareta, pá).|
| RF11 | fertilazação| O sistema deve permitir agricultura com uso de adubo.|

## Requisitos não funcionais

| ID | Nome | Descrição |
| :--- | :--- | :--- |
| RNF01 | Tecnologia | Implementar em Java ou Unreal (POO conforme Sommerville). |
| RNF02 | Estética Visual | Processar gráficos em Pixel Art (Sprites de baixa resolução). |
| RNF03 | Integridade | Manter histórico de versões e documentação íntegra no GitPages. |
