# Glossário

## Introdução

1. Introdução
Este documento tem como objetivo definir os termos técnicos, conceitos de jogo e siglas utilizados na documentação do projeto. A padronização destes termos visa alinhar o entendimento entre os membros da equipe e os stakeholders, garantindo que a comunicação seja clara e sem ambiguidades.

## 2. Definições

| Termo | Definição |
| :--- | :--- |
| **After Effects / Premiere** | Softwares da Adobe utilizados, respectivamente, para criação de animações e edição de vídeo na produção do demo do projeto. |
| **Avatar 2D** | Personagem controlado pelo jogador, representado graficamente em duas dimensões. No projeto, possui sprites distintos para os estados de movimento, coleta e transporte de itens. |
| **Blueprint** | Sistema visual de programação por nós (node-based) do Godot, utilizado para implementar a lógica do jogo sem escrita direta de código. Equivalente funcional ao Blueprint do Unreal Engine, adaptado ao contexto da engine escolhida. |
| **Crafting** | Mecânica de jogo que permite a criação de novos itens a partir da combinação de materiais coletados. No projeto, ocorre por meio da maquininha de reciclagem, seguindo receitas predefinidas que descontam os materiais do inventário. |
| **Design Sprint** | Metodologia ágil de cinco dias utilizada para validar ideias e resolver problemas por meio de prototipagem e testes rápidos. Adotada como processo central de desenvolvimento deste projeto. |
| **Elicitação** | Conjunto de técnicas utilizadas para descobrir e extrair requisitos de software a partir de diversas fontes (atas, reuniões, observação). |
| **FURPS+** | Acrônimo para um modelo de classificação de atributos de qualidade de software (Functionality, Usability, Reliability, Performance, Supportability). |
| **GitHub Pages** | Serviço de hospedagem estático do GitHub utilizado para publicar a documentação do projeto, incluindo artefatos do design sprint, gravações de reuniões e o histórico de contribuições por membro. |
| **Godot** | Engine de jogo multiplataforma e de código aberto utilizada como base tecnológica para o desenvolvimento deste projeto, escolhida pela simplicidade de uso e suporte nativo a jogos 2D. |
| **Grid** | Divisão do mapa em células quadradas de tamanho fixo. Define a escala dos sprites, a lógica de posicionamento de objetos e a área de irrigação das plantações. |
| **Inventário** | Sistema de gerenciamento de itens que armazena os recursos coletados pelo avatar durante a exploração do mapa. |
| **Maquininha de reciclagem** | Objeto único do mapa responsável por receber materiais do inventário e transformá-los em novos itens segundo as receitas de crafting disponíveis. |
| **MoSCoW** | Método de priorização utilizado para definir a importância de cada requisito (Must have, Should have, Could have, Won't have). |
| **On click** | Padrão de interação em que uma ação do jogo (como coleta ou uso de ferramenta) é acionada pelo clique do jogador, disparando a animação correspondente. |
| **Pixel Art** | Forma de arte digital em que as imagens são criadas e editadas ao nível do pixel, resultando em estética visual retro e de baixa resolução. No projeto, os sprites são produzidos manualmente no Piskel, sem uso de geração por IA. |
| **Piskel** | Ferramenta online gratuita de criação de pixel art e animações frame a frame, adotada no projeto para produção dos sprites exportados em PNG ou GIF para uso no Godot. |
| **Rich Picture** | Artefato visual informal utilizado no design sprint para representar a visão de cada membro sobre o sistema, suas relações e os problemas a resolver, sem seguir um formalismo rígido. |
| **Sprite** | Imagem bidimensional que representa visualmente um personagem ou objeto no jogo. No projeto, cada estado do avatar (andando, coletando, segurando item) possui um sprite dedicado, criado no Piskel e exportado para o Godot. |
| **WASD** | Esquema de controle que utiliza as teclas W (frente), A (esquerda), S (trás) e D (direita) para a movimentação do avatar. |
