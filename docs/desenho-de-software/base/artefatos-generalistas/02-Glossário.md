# Glossário

## Introdução

1. Introdução
Este documento tem como objetivo definir os termos técnicos, conceitos de jogo e siglas utilizados na documentação do projeto. A padronização destes termos visa alinhar o entendimento entre os membros da equipe e os stakeholders, garantindo que a comunicação seja clara e sem ambiguidades.

## 2. Definições

| Termo | Definição |
| :--- | :--- |
| **Avatar 2D** | Personagem controlado pelo jogador, representado graficamente em duas dimensões. |
| **Crafting** | Mecânica de jogo que permite a criação de novos itens a partir da combinação de materiais coletados ou reciclados. |
| **Elicitação** | Conjunto de técnicas utilizadas para descobrir e extrair requisitos de software a partir de diversas fontes (atas, reuniões, observação). |
| **FURPS+** | Acrônimo para um modelo de classificação de atributos de qualidade de software (Functionality, Usability, Reliability, Performance, Supportability). |
| **Godot** | Engine de jogo multiplataforma e de código aberto utilizada como base tecnológica para o desenvolvimento deste projeto. |
| **Inventário** | Sistema de gerenciamento de itens que armazena os recursos coletados pelo avatar durante a exploração do mapa. |
| **MoSCoW** | Método de priorização utilizado para definir a importância de cada requisito (Must have, Should have, Could have, Won't have). |
| **Pixel Art** | Forma de arte digital onde as imagens são criadas e editadas ao nível do pixel, resultando em uma estética visual retro e de baixa resolução. |
| **Sprite** | Imagem bidimensional ou animação integrada a uma cena maior, representando objetos ou personagens no jogo. |
| **WASD** | Esquema de controle que utiliza as teclas W (frente), A (esquerda), S (trás) e D (direita) para a movimentação do avatar. |
