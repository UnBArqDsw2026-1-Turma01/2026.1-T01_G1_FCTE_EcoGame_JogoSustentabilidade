# Visão Geral da Elicitação e Design

## Introdução
<p align="justify">Esta seção consolida a estratégia do projeto, integrando criatividade e rigor técnico através da metodologia Design Sprint. O escopo foi construído de forma iterativa para garantir rastreabilidade entre a ideia inicial e o produto final.</p>

### Fluxo de Desenvolvimento
O escopo do projeto foi definido através de três pilares fundamentais:

- [Brainstorming](../elicitacao/brainstorming.md): Exploração criativa de tecnologias, temas de sustentabilidade e referências visuais para alinhar as intenções do grupo.

- [Especificação e Priorização de Requisitos (Formalização)](../elicitacao/requisitos.md): Tradução técnica das necessidades em requisitos funcionais e não funcionais (FURPS+) priorizados via MoSCoW.

- [Entrevistas de Validação (Convergência)](../elicitacao/entrevistas.md): Testes com usuários reais para transformar suposições em dados, refinando mecânicas de crafting e usabilidade.
